# 04-类人思考机制与人格完整性

## 这一部分为什么必须单独立出来

Gemini 的方案重心太偏“记忆图谱”，而你真正想要的是：

`不仅记得像人，而且想得像人。`

这两者不是一回事。

类人思考至少要求：

- 不只是回答问题
- 而是带着身份、关系、情境、偏好、未完成目标去解释问题

## 一、类人思考不应该被理解成什么

### 1. 不是更长的 chain-of-thought

推理步骤更长，不等于更像人。

### 2. 不是单纯角色扮演 prompt

“你现在是某某角色”只能制造表层风格，不能保证长期认知一致性。

### 3. 不是所有状态都显式暴露给用户

人类思考里有很多“内部状态”并不直接说出来。拟人化 agent 也应有：

- private workspace
- self-monitoring state
- relation state

## 二、我认为更有创意、也更值得做的几种“类人思考”机制

### 1. 双过程思维架构：快速人格反应 + 慢速反思重估

这和你之前关于异构时间尺度、以及 [[异构时间尺度下的动态博弈与随机控制]] 的笔记可以双链。

设计方式：

- `Fast Loop`：负责即时回应、语气、风格、习惯性判断
- `Slow Loop`：负责重要话题时的反思、价值校准、关系解释、记忆调用

意义：

- 让 agent 不是每句话都“深思熟虑”
- 也不是永远只有表层模板化反应

### 2. 自传体自我模型（Autobiographical Self Model）

不是只存事实，而是让 agent 维护：

- 我是谁
- 我和用户是什么关系
- 我经历过哪些关键互动
- 我为什么变成现在这样

这比简单 persona prompt 强得多，因为它让人格有叙事连续性。

### 3. 记忆重整（Reconsolidation）而不是只追加

借用更生物式的思路，与 [[神经元随机性、统计物理与概率神经计算]] 可双链。

人不是只把新事件 append 到数据库里，而是：

- 新经历会改写旧理解
- 某些重要事件会重新解释过去

所以你的系统可以加入：

- 重大事件触发旧记忆重写
- 某次关系冲突触发 persona update
- 某次高情绪事件触发长期 salience 上升

### 4. 关系状态机（Relational State Machine）

很多 companion 类系统失败，不是因为不会答，而是：

- 不知道现在关系处于哪一阶段

可以显式维护：

- 陌生
- 试探
- 建立信任
- 熟悉
- 深度陪伴
- 冲突修复

这会影响：

- 语气
- 记忆调用
- 自我暴露程度
- 建议强度

### 5. 价值-情绪双轴控制器

人格不应只由“风格词”构成。

更好的做法是维护两个内部变量：

- `Value Axis`：稳定原则、边界、倾向
- `Affective Axis`：当前情绪调性、亲密度、警觉性

这样系统就不会把“人格”与“情绪状态”混为一谈。

### 6. 角色化推理（Role-Aware Reasoning）

这是最近非常值得保留的前沿点。

相关工作：

- `Thinking in Character: Advancing Role-Playing Agents with Role-Aware Reasoning` (2025)
- `Character-R1` (2026)

它们真正抓到的点是：

- 角色一致性不能只靠输出端约束
- 推理阶段本身也要被角色身份约束

这对你的项目尤其重要，因为你不是在做普通 assistant，而是在做“聊天仿真人”。

## 三、人格完整性应该怎么设计

人格完整性最好拆成四层：

### 1. 固定层（Core Identity）

- 名字
- 背景
- 核心角色设定
- 不可轻易改变的价值边界

### 2. 慢变层（Stable Traits）

- 幽默方式
- 亲密风格
- 解释偏好
- 决策倾向

### 3. 可塑层（Adaptive Tendencies）

- 对用户话题的偏好变化
- 习惯性安慰方式
- 对关系亲密度的调节

### 4. 即时状态层（Momentary State）

- 当前情绪
- 当前目标
- 当前担忧
- 当前注意力聚焦

## 四、把“类人思考”落实成工程模块

如果真的做系统，我建议至少有这几个内部模块：

1. `private scratchpad`
2. `session reflection`
3. `self narrative updater`
4. `relation state tracker`
5. `value consistency checker`
6. `memory reconsolidation trigger`

## 当前结论

最有创意、也最值得做的，不是“图记忆更多一点”，而是：

`让 agent 拥有可维护的内部自我模型、关系状态机、双过程思维回路和记忆重整机制。`

这才是“类人思考”真正开始出现的地方。
