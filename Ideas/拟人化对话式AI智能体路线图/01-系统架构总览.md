# 01-系统架构总览

## 总架构原则

如果目标是长期拟人化对话体，系统不能只由一个 LLM 加一个向量数据库构成。更合理的架构应该至少拆成五层：

1. `基础语言/推理层`
2. `工作记忆层`
3. `长期记忆层`
4. `人格与自我模型层`
5. `反思与策略控制层`

Gemini 把重点放在“动态拓扑记忆图谱”，这是其中一层，但不是全部。

## 1. 基础语言 / 推理层

职责：

- 语言理解与生成
- 局部推理
- 风格化表达

当前最现实方案：

- 直接使用强基座模型，而不是从零训练
- 将“拟人化能力”更多放在系统架构，而不是底模预训练上

原因：

- 语言能力、常识能力、风格迁移能力，当前最经济的做法就是借用现成 frontier LLM
- 真正稀缺的是“长期人格连续性与记忆治理”

## 2. 工作记忆层

职责：

- 处理当前会话中的即时状态
- 保存当前对话目标、情绪氛围、未完成意图、最近结论

这层不应等同于“整个上下文窗口”，而应是：

- 被系统显式维护的短期状态区
- 包含当前会话摘要、当前心理状态、最近关注点

这与 MemGPT / Letta 的 memory tier 思路是相通的。

## 3. 长期记忆层

这层是 Gemini 路线中最有价值的部分，但要做扩展。

不应只是一种存储，而应分成至少四类：

### 3.1 事件记忆（episodic memory）

- 某天说了什么
- 发生了什么具体事件
- 某段关系是如何推进的

### 3.2 语义记忆（semantic memory）

- 关于用户的稳定事实
- 关于世界的稳定知识
- 关于智能体自身的设定事实

### 3.3 人格记忆（persona memory）

- 智能体自我定义
- 稳定价值观
- 风格偏好
- 底层行为原则

### 3.4 反思记忆（reflective / metacognitive memory）

- 它对自己行为的总结
- 它认为哪些模式有效
- 它对关系演化的内部解释

Gemini 的 GNN 图谱更适合其中的：

- 事件关联
- 因果 / 时序 / 人物关系结构

但不一定适合单独承担全部长期记忆功能。

## 4. 人格与自我模型层

这是 Gemini 明显说少了的一层。

拟人化对话体必须显式维护：

- `Agent Profile`：固定身份设定
- `Agent Traits`：可缓慢演化的偏好和性格倾向
- `Values / Boundaries`：价值观与边界条件
- `Self-Narrative`：对自己是谁、和用户是什么关系的叙事

这一层更接近：

- Memory OS of AI Agent 中的 `Agent Persona`
- Letta 的 `persona memory block`

如果这一层缺失，系统即便“有记忆”，也只会像数据库，不会像一个人。

## 5. 反思与策略控制层

这层决定系统是不是“只是会回忆”，还是“会思考”。

职责：

- 选择哪些记忆值得写入长期层
- 选择哪些记忆需要压缩、重整、遗忘
- 在冲突信息出现时决定如何修正自我叙事
- 对重要互动做 session-level reflection

这里与：

- Generative Agents 的 `memory + reflection + planning`
- Reflective Memory Management 的 `prospective / retrospective reflection`

直接相关。

## 6. 为什么 Gemini 的 GNN 路线不能成为“当前唯一重点”

GNN / KG / Neo4j 可以做：

- 结构化关系存储
- 多跳关联检索
- 因果和时间边的显式建模

但当前阶段的拟人化对话体，真正更关键的不是“是否用了图”，而是：

1. 记忆写入标准
2. 记忆检索时机
3. 人格状态更新规则
4. 反思生成与记忆重整
5. 风格一致性与身份不漂移

换句话说：

`GNN 是可选增强层，不是先验唯一主轴。`

## 当前推荐架构

最适合现阶段的形态是：

`强基座 LLM + 分层记忆系统 + 人格状态层 + 反思控制层 + 可选图谱增强`

而不是：

`先上 Neo4j + GNN，再谈人格。`
