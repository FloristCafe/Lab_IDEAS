# 06-阶段路线图与MVP

## 总原则

不要一开始就做：

- GNN 全图人格传播
- 分布式微服务
- 神经形态硬件迁移

这些都太早。

现阶段最重要的是：

`先做一个长期不失忆、不严重漂人格、具备基础反思机制的拟人化对话体。`

## 阶段 0：定义人格与记忆规范

### 目标

在任何代码之前，先定义数据结构。

### 需要产出

1. Agent Profile schema
2. User Profile schema
3. Episode memory schema
4. Semantic memory schema
5. Reflection memory schema
6. Trait update rules
7. Forgetting / reinforcement policy

### 为什么先做这个

如果 schema 不清晰，后面系统只会变成一堆 prompt 和脏数据。

## 阶段 1：单体 MVP

### 目标

做出一个单进程、单用户、长期对话体原型。

### 推荐最小模块

1. 强基座 LLM
2. 会话工作记忆
3. 长期记忆数据库
4. 人格状态文件
5. 会话后反思器
6. 检索与写入策略

### 这一阶段不做什么

- 不做复杂分布式
- 不做图神经网络训练
- 不做自动互联网学习

### 成功标准

- 跨 50-100 次会话保持基本角色一致
- 能正确记住核心用户事实
- 会话结束后能生成反思并影响后续对话
- 不出现明显人格断裂

## 阶段 2：结构化记忆增强

### 目标

给 MVP 加上结构化长期记忆层。

### 可加内容

1. 事件-人物-偏好图谱
2. 多跳关系检索
3. 重要事件的因果链组织
4. 冲突记忆的版本管理

### 这里再考虑图

Gemini 提到的 Neo4j / GNN 可以在这一步介入，但建议优先：

- 先用图数据库做关系存储
- 再决定是否真的需要 GNN

因为大多数早期收益来自：

- 结构化索引
- 更好的检索与关联

而不是端到端图表示学习。

## 阶段 3：类人思考增强

### 目标

让系统从“会长期记忆的聊天机器人”升级为“有内部认知痕迹的拟人化对话体”。

### 推荐加入

1. 双过程回路
2. relation state tracker
3. self narrative updater
4. value consistency checker
5. memory reconsolidation

### 验证什么

- 它是否会在重要话题上变得更“像一个持续存在的人”
- 它是否能在冲突中保持自我连续性而不是乱漂

## 阶段 4：自主学习与任务驱动知识获取

### 目标

在你允许的边界下，逐步让 agent 自主补充知识。

### 关键前提

必须先有：

- 记忆治理
- 来源标注
- 错误经验修正
- 用户可编辑与可回滚机制

否则自主学习只会污染人格与记忆。

## 当前 MVP 最该优先实现的功能

### 必做

1. 显式人格层
2. 分层记忆
3. 记忆写入规则
4. 会话级反思
5. 简单遗忘策略

### 暂缓

1. GNN 人格传播
2. C++ 图算法优化
3. 微服务拆分
4. 仿脑硬件映射

## 当前一句话实施建议

第一版最优路线不是“图神经网络优先”，而是：

`先做一个 stateful、能反思、有人格状态层的单体 companion agent；图谱和更重的认知结构，作为第二阶段增强。`
